Agent 对接指南

这个聊天页面暴露了一套 window API，任何能控制浏览器的 AI Agent 都能对接。核心流程：读用户消息 → 生成 JSON Spec → 渲染到气泡。

能力要求：Agent 必须能在页面上下文执行 JavaScript（Chrome DevTools MCP evaluate_script、Playwright page.evaluate、CDP 等）。如果只能点击和输入，它只能发送用户消息，不能调用 renderVizInMsg() 完成渲染。

Bridge API（页面暴露的全局函数）

API说明返回
window._pendingMsg用户发送的最新原始消息（保留换行；读完不会自动清）string | null
window.getPendingMessage()读取带 id / rawText / createdAt 的待处理消息；优先用它，不要从 DOM 反读用户输入object | null
window.markPendingHandled()标记当前待处理消息已处理，并写入 __lastHandledMessagevoid
window.createAiMsg()创建 AI 消息气泡（带流式光标），返回消息 IDstring (id)
window.streamText(id, text)往 AI 气泡追加文字void
window.finishText(id)结束流式输出（移除光标）void
window.renderVizInMsg(id, spec, options?)渲染静态 Vizual JSON Spec 到气泡；兼容旧写法，内部会保存可编辑 artifactboolean
window.renderArtifactInMsg(id, artifact, options?)渲染完整 VizualArtifact；用于历史恢复、主题、版本、targetMapVizualArtifact | null
window.getLastArtifact()读取最近一次渲染的 artifact，包含 spec、targetMap、versions、exportsVizualArtifact | null
window.updateArtifactInMsg(ref, patches, options?)用 Vizual typed patches 修改 artifact 并渲染到新的 AI 气泡；仅预览/调试时传 { mode: 'replace' } 原地替换。兼容常见 JSON Patch，但 Agent 应优先使用 typed patches。VizualArtifact | null
window.exportArtifact(ref, options?)导出当前 artifact 并记录 export 元数据；options.format 支持 png/pdf/csv/xlsxPromise<ExportRecord | null>
window.renderInteractiveVizInMsg(id, config)渲染 FormBuilder 控件 + 实时预览；Agent 作为宿主提供 makeSpec(state)boolean
window.updateInteractiveVizInMsg(ref, patch, options?)以稳定 API 更新实时预览 state；自动化测试不要模拟 React DOM 输入事件object | null
window.getInteractiveVizState(ref?)读取实时预览 state、artifact、lastPreviewSpec 和 renderCountobject | null
window.getVizualConversationState()读取稳定的消息、artifact、控件和渲染历史状态；自动化测试不要猜 DOM classobject
window.getVizualDebugState()读取 pending message、artifact registry、DocView session 和最近渲染记录object
window.renderDocViewInMsg(id, config)渲染带审阅 bridge 的 DocView；用于可批注报告、长文共建、修订循环VizualArtifact | null
window.getDocViewReviewState(ref?)读取 DocView sections、threads、revisionProposals、review eventsobject | null
window.createDocViewThread(ref, input)创建 DocView 批注线程；可传完整 anchor，也可用 sectionId/sectionIndex + selectedText 让宿主补全 anchorAnnotationThread | null
window.createDocViewRevision(ref, input)Agent 根据已提交批注创建修订提案；用户可在面板点击应用RevisionProposal | null
window.applyDocViewRevision(ref, proposalId)应用修订提案，更新 DocView sections 和 artifactRevisionProposal | null

window.getVizualConversationState()、window.__lastVizualRender、window.__renderHistory、window.__vizualArtifacts 会记录最后一次静态/交互渲染的 spec、targetMap、版本和导出信息，便于 QA 检查 Agent 是否真的使用了用户数据。

bubbleWidth 可以省略，页面会按组件类型自动推断。KPI/迷你图偏窄，普通图表偏宽，Grid/Split/DocView/Sankey/Radar/FormBuilder/DataTable 默认 full。

// 完整流程
const pending = window.getPendingMessage(); // 1. 读取原始输入（保留换行）
const id = window.createAiMsg();            // 2. 创建气泡
window.streamText(id, '正在生成图表...');     // 3. 流式文字
window.finishText(id);                       // 4. 结束文字
window.renderVizInMsg(id, spec);             // 5. 渲染图表（自动推断气泡宽度）
window.markPendingHandled();                 // 6. 清除已读消息

历史追问与修改

用户后续说“这张图改成折线图，只看华东区，再导出成 PDF/图片/表格”时，不要重新猜测上一张图。先读 artifact，再用 Vizual typed patch 修改目标组件。默认会生成新的 AI 气泡，保留历史消息；只有临时预览才用 { mode: 'replace' } 原地替换。

const artifact = window.getLastArtifact();
const updated = window.updateArtifactInMsg(artifact.id, [
  { type: 'changeChartType', targetId: 'element:chart', chartType: 'LineChart' },
  { type: 'filterData', targetId: 'element:chart', field: 'region', values: '华东' },
  { type: 'limitData', targetId: 'element:chart', limit: 8 },
], { answerText: '已按华东区改成折线图。' });
const exportRecord = await window.exportArtifact(updated.id, { format: 'pdf', filename: 'east-china-chart' });
const tableRecord = await window.exportArtifact(updated.id, { format: 'xlsx', filename: 'east-china-data' });

实时 adjust-preview 场景

实时可调图表不是纯 JSON spec。Agent 必须作为宿主调用 renderInteractiveVizInMsg()，用 FormBuilder 绑定 /controls，再由 makeSpec(state) 生成右侧预览图表。

控件必须按当前图表能力联动：例如横向/堆叠只属于柱状图，切换到折线图或组合图时要用 dependsOn / showWhen 隐藏，并且在 makeSpec(state) 里不要传入非法 props。

const id = window.createAiMsg();
window.streamText(id, '可以直接调参数查看结果。');
window.finishText(id);

window.renderInteractiveVizInMsg(id, {
  initialState: {
    controls: {
      chartType: 'bar',
      points: 8,
      orientation: 'vertical',
      stacked: false,
      smooth: false,
      secondaryMetric: 'growth',
      brandColor: '#ff6b35',
    },
  },
  bubbleWidth: 'full',
  designMd: 'Primary: #ff6b35\nBackground: #0f1117',
  controlsSpec: {
    root: 'controls',
    elements: {
      controls: {
        type: 'FormBuilder',
        props: {
          type: 'form_builder',
          value: { $bindState: '/controls' },
          fields: [
            {
              name: 'chartType',
              label: '图表类型',
              type: 'select',
              options: [
                { label: '柱状图', value: 'bar' },
                { label: '折线图', value: 'line' },
                { label: '组合图', value: 'combo' },
              ],
            },
            { name: 'points', label: '数据点', type: 'slider', min: 3, max: 14 },
            {
              name: 'orientation',
              label: '柱状方向',
              type: 'select',
              options: [
                { label: '纵向', value: 'vertical' },
                { label: '横向', value: 'horizontal' },
              ],
              dependsOn: 'chartType',
              showWhen: 'bar',
            },
            { name: 'stacked', label: '堆叠柱状图', type: 'switch', dependsOn: 'chartType', showWhen: 'bar' },
            { name: 'smooth', label: '平滑折线', type: 'switch', dependsOn: 'chartType', showWhen: 'line' },
            {
              name: 'secondaryMetric',
              label: '组合图折线指标',
              type: 'select',
              options: [
                { label: '增长率', value: 'growth' },
                { label: 'ARPPU', value: 'arppu' },
              ],
              dependsOn: 'chartType',
              showWhen: 'combo',
            },
            { name: 'brandColor', label: '品牌色', type: 'color' },
          ],
        },
        children: [],
      },
    },
  },
  applyTheme: (state, Vizual) => {
    Vizual.loadDesignMd('Primary: ' + state.controls.brandColor, { apply: true });
  },
  makeSpec: (state) => {
    const points = Number(state.controls.points || 8);
    const data = Array.from({ length: points }, (_, i) => ({
      day: 'D' + (i + 1),
      value: 20 + i * 5,
      growth: 8 + i * 1.8,
      arppu: 4.5 + i * 0.35,
    }));
    const chartType = state.controls.chartType;
    const chart =
      chartType === 'line'
        ? { type: 'LineChart', props: { type: 'line', title: '实时预览', x: 'day', y: 'value', data, smooth: !!state.controls.smooth }, children: [] }
        : chartType === 'combo'
          ? { type: 'ComboChart', props: { type: 'combo', title: '实时预览', x: 'day', y: ['value', state.controls.secondaryMetric || 'growth'], data }, children: [] }
          : { type: 'BarChart', props: { type: 'bar', title: '实时预览', x: 'day', y: 'value', data, stacked: !!state.controls.stacked, horizontal: state.controls.orientation === 'horizontal' }, children: [] };
    return {
      root: 'chart',
      elements: {
        chart: {
          ...chart,
        },
      },
    };
  },
});

DocView 批注循环

只有“需要用户批注、提交、Agent 修订、再应用”的报告才用 DocView。普通 dashboard 不要用 DocView。

const id = window.createAiMsg();
window.streamText(id, '我生成了一份可批注报告。');
window.finishText(id);
const artifact = window.renderDocViewInMsg(id, {
  sections: [
    { id: 'title', type: 'heading', content: '经营分析报告', level: 1 },
    { id: 'summary', type: 'text', content: '收入下滑主要来自活跃用户下降。' },
  ],
  showPanel: true,
});

// 用户提交批注后，Agent 读取待处理线程
const state = window.getDocViewReviewState(artifact.id);
const submitted = state.threads.filter(t => t.status === 'submitted');

// Agent 创建修订提案
window.createDocViewRevision(artifact.id, {
  fromThreadIds: submitted.map(t => t.id),
  summary: '补充收入下滑原因',
  patches: [
    { op: 'updateSection', sectionId: 'summary', updates: { content: '收入下滑主要来自活跃用户下降，同时高价值用户留存推高 ARPPU。' } },
  ],
});

Claude Code（Chrome DevTools MCP）

安装 chrome-devtools-mcp，Claude Code 自动获得 mcp__chrome-devtools__evaluate_script 工具：

// 监听用户输入
const msg = await mcp__chrome-devtools__evaluate_script({
  function: '() => window.getPendingMessage()'
});

// 回复
await mcp__chrome-devtools__evaluate_script({
  function: 'async () => {\n' +
    'const id = window.createAiMsg();\n' +
    'window.streamText(id, "这是回复文字");\n' +
    'window.finishText(id);\n' +
    'const resp = await fetch("/validation/my-spec.json");\n' +
    'const spec = await resp.json();\n' +
    'const ok = window.renderVizInMsg(id, spec);\n' +
    'window.markPendingHandled();\n' +
    'return ok;\n' +
  '}'
});

Codex / Gemini / 通用方案（Playwright）

任何能跑 Node.js 的 Agent 都能用 Playwright 控制浏览器：

import { chromium } from 'playwright';

const browser = await chromium.launch({ headless: false });
const page = await browser.newPage();
await page.goto('http://localhost:8765/validation/vizual-test.html');

// 等用户输入 — 轮询 getPendingMessage()
const pending = await page.evaluate(() => window.getPendingMessage());
const msg = pending?.rawText || pending?.text;

// 回复
await page.evaluate((spec) => {
  const id = window.createAiMsg();
  window.streamText(id, '这是回复文字');
  window.finishText(id);
  window.renderVizInMsg(id, spec);
  window.markPendingHandled();
}, mySpec);

纯 HTTP 轮询（无浏览器控制）

如果 Agent 完全无法控制浏览器，可以让页面自己轮询一个后端：

// 在页面 script 里加一个自动轮询
setInterval(async () => {
  const pending = window.getPendingMessage();
  if (pending) {
    const resp = await fetch('http://localhost:3000/chat', {
      method: 'POST',
      body: JSON.stringify({ message: pending.rawText || pending.text }),
    });
    const { spec } = await resp.json();
    const id = window.createAiMsg();
    window.finishText(id);
    window.renderVizInMsg(id, spec);
    window.markPendingHandled();
  }
}, 2000);
