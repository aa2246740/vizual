这个矩阵在验收什么

每个 Part 都是一个协议入口的最小真实用法：先喂给 VizualNativeCore，再由同一个原生 renderer 渲染，最后检查可见内容、交互状态、图表画布和错误。

  1. A2UI 原生组件图片、表单、按钮、数据绑定和 action。
  2. A2UI 模板 children列表模板按数据模型展开。
  3. AG-UI 流式事件消息、state patch、tool args/result 拼成原生界面。
  4. AGenUI 扩展Chart/Table/Markdown/RichText/Web/Lottie/Carousel。
  5. AGenUI 全 catalog25 个 catalog 组件至少有可见表征。
  6. 上游交付兼容A2UI MIME、MCP/A2A resource、AG-UI render_a2ui 新参数。
  7. AG-UI 全事件名33 个 EventType 进入同一个 native core。
core 不做布局/设计门禁，只验证协议语义、渲染可见、交互可触发 runtime loading
