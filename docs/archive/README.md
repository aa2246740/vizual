# Vizual Documentation Archive

This directory keeps historical planning, research, and acceptance artifacts out
of the public documentation path.

Use these files for archaeology, regression investigation, or understanding why
an older decision was made. Do not treat them as current API or integration
contracts unless a current top-level document links to a specific section.

## Contents

- `acceptance/`: historical cold-start and manual acceptance prompts.
- `adr/`: early decision notes. These are brief and may be superseded by
  `docs/INTEGRATION.md` or `docs/AI-INTEGRATION.md`.
- `handoffs/`: one-off agent handoff notes.
- `plans/`: dated implementation plans and design explorations.
- `prd/`: dated product requirement snapshots.
- `research/`: external library reviews and protocol research.
